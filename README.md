# TV Script Generation with RNNs

A deep learning project that uses **Recurrent Neural Networks (RNNs)** to generate brand-new, fake *Seinfeld* TV scripts. Built as part of Udacity's Deep Learning Nanodegree program.

---

## 📌 Project Overview

This project trains an RNN on scripts from all 9 seasons of *Seinfeld* to learn the patterns of dialogue, character voices, and scene structure. Once trained, the model generates original "fake" TV scripts that mimic the style of the show.

The project covers the full NLP pipeline: loading and preprocessing raw text, building a vocabulary, creating batched data loaders, designing and training an LSTM-based RNN, and sampling new text from the trained model.

## 🧠 Model Architecture

### Data Preprocessing
- Loads raw `Seinfeld_Scripts.txt` (all lowercase, dialogue separated by newlines)
- Builds a word-level vocabulary (word → integer lookup tables)
- Creates batched, sequential input/target pairs for training

### RNN Model (PyTorch)
The model is implemented as a custom `RNN` class using `torch.nn.Module`:

- **Embedding layer** — maps word indices to dense vector representations
- **LSTM layers** — 2–3 stacked LSTM layers to capture long-range dependencies in dialogue
- **Fully connected output layer** — maps hidden state to vocabulary-sized logits
- **Hidden state initialization** — supports batch-first formatting

Key design decisions:
- **LSTM vs GRU**: LSTM was found to produce better results
- **Layers**: 2–3 layers recommended for sufficient capacity
- **Hidden dimension**: Typically set larger than the embedding dimension (e.g., embedding=300, hidden=512)
- **Sequence length**: Shorter sequences (e.g., 10–50 tokens) train significantly faster

### Text Generation
After training, new scripts are generated by:
1. Seeding the model with a starting word/token
2. Sampling the next word from the output probability distribution
3. Feeding the predicted word back in as the next input
4. Repeating until the desired script length is reached

## ⚙️ Hyperparameters

| Parameter         | Suggested Range     |
|------------------|---------------------|
| Sequence Length   | 10 – 50             |
| Batch Size        | 32 – 128            |
| Embedding Dim     | 200 – 500           |
| Hidden Dim        | 256 – 1024          |
| Number of Layers  | 2 – 3               |
| Learning Rate     | 0.0001 – 0.01       |
| Epochs            | 10 – 30+            |

> **Note:** Hyperparameter tuning requires patience — each epoch can take 15+ minutes without a GPU.

## 💡 Possible Improvements

- Use a larger portion (or all) of the Seinfeld dataset for richer training
- Experiment with GRU cells as an alternative to LSTM
- Apply word embeddings pre-trained on a large corpus (e.g., GloVe, Word2Vec)
- Implement temperature-based sampling for more creative (or conservative) generation
- Try a Transformer-based architecture (e.g., GPT-2 fine-tuning) for higher-quality text
- Extend to other TV shows by swapping in different script datasets
